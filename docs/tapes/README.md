# Demo media

How the README's demo assets are regenerated. All of them render the cat demo
tree under `docs/demo/` (the *Millions of Cats* field guide), so they stay in
sync with that content.

## TUI gifs — `docs/assets/fur-*.gif`

Recorded with [vhs](https://github.com/charmbracelet/vhs) from the `.tape`
files here, then optimized with [gifsicle](https://www.lcdf.org/gifsicle/).

```bash
# requires: vhs (needs ttyd + ffmpeg), gifsicle
for t in hero search data themes; do
  vhs docs/tapes/$t.tape                                   # -> docs/assets/fur-$t.gif
  gifsicle -O3 --lossy=80 docs/assets/fur-$t.gif -o /tmp/o.gif && mv /tmp/o.gif docs/assets/fur-$t.gif
done
```

Notes:
- `Output` paths in the tapes are **relative** — run vhs from the repo root.
- vhs renders text only: it **cannot** capture terminal image protocols
  (Sixel/Kitty/iTerm2), so inline images never appear in a gif. The TUI shows a
  text info card for images by design anyway (alt-screen incompatibility).
- Light themes can't be captured in vhs (the OSC-11 background query confuses
  its terminal); the `themes` tape cycles dark palettes only.

## Web screenshots — `docs/assets/web-*.webp`

Regenerated by a headless browser. The actual cat illustrations render here
(browsers decode webp), which is the web-mode counterpart to the gifs.

```bash
make capture-web        # serve demo -> drive browser -> PNG -> webp -> cleanup
```

Notes:
- Needs a Chrome/Chromium binary. `e2e/capture` auto-detects the standard names
  plus the Playwright/rod download caches, so no system Chromium is required
  (Ubuntu 24.04 ships only a snap, which chromedp can't drive). Override with
  `FUR_CHROME=/path/to/chrome`.
- d3 is vendored (`internal/web/static/d3.v7.min.js`), so the graph renders
  fully offline. Markdown pages with Mermaid diagrams still fetch Mermaid from
  the jsdelivr CDN, so capturing one of those would need network.
- The capture tool lives in the `e2e` module so its browser dependency never
  touches the main module.
